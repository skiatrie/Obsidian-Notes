- Every being has slightly different characteristics, yet as a species, there are many consistently inherited attributes and behaviors

When it comes to Object-Oriented programming:
	- classes can inherit data and methods from existing classes
	**Inheritance** - Classes can inherit behavior and attributes from preexisting classes. Create a new class based on how much it differs from a class that has already been developed and tested

- creating a class by inheriting another class, the new class automatically contains the methods and data fields of the original class

The `Employee` class

``` Java
public class Employee
private int id;
private double salary;
public int getId() 
	{
	return id;
	}
public double getSalary()
	{
	return salary;
	}
public int setId(int idNum)
	{
	id = idNum;
	}
public double getSalary(double sal)
	{
	salary = sal;
	}
}
```

- then you can create:
``` Java
Employee receptionist = new Employee();
Employee teacher = new Employee();
```

- these are new employee [objects] that can potentially have differing ID numbers and salaries
- Since they are `Employee` objects, they will have some ID number and salary

Benefits of using inheritance:
- save time since you are reusing methods that already exist
- reduce errors because methods have been used and tested before
- reduce new learning required to use new class since alr familiar with old class
- develop new classes quickly and effectively

## Terminology

base class - class that is used as a basis for inheritance
derived class - class created that inherits from a base class

- a **derived** class always [is a(n)] case or example of the base class
	a `tree` class can be a base to an `evergreen` class
	an `evergreen` "is a" `tree` - so `tree` is the base class
	*Not the other way around!*
- "is a" is used to described an object's relation to a class

Not to be confused with...
"has a" - **containment** - when one class contains fields that are members of another class
- **composition** - a class contains one of more members of another class
- members would not exist without object containing them

- **aggregation** - class contains one or more members of another class, and those members will continue to exist without the class that contains them

superclass / subclass - synonyms for *base* class and *derived* class
parent class / child class - same synonym
`evergreen` is the subclass of the `tree`superclass

- you can also state them in the "child-parent" order
Evergreen Tree, evergreen is the child, tree is  the parent
