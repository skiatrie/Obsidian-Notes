When superclass data fields/methods are not appropriate for the subclass objects, override parent class members.

**override** - use the child's version instead of the parent's version
[polymorphism] - using the same method name to indicate different implementations
- type of action depends on the associated object
- if not supported by a programming language, the lang is not object-oriented

Example:

Each employee may have different pay periods. Create a subclasses of `Employee` : `HourlyEmployee` and `ContractEmployee`

Using `displayRateofPay` method in each class, with their individual data fields, each class with use the appropriate method

- Each subclass method overrides any method in the superclass that has the *same* name and parameter list
	- same name but diff parameter list: subclass method does not override
	it **overloads** the parent class method, any subclass obj has access to both versions


"+" operand is polymorphic
methods with same name / diff parameter lists are polymorphic - method calls operate differently depeneding on arg
